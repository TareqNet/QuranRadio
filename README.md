# دليل توثيق الإضافة (Quran Radio v2.0)

هذا المستند التقني يوفر شرحاً مفصلاً لكافة الميزات والتحسينات المعمارية التي تم إنجازها في التحديث الشامل لإضافة "إذاعة القرآن الكريم" على متصفح جوجل كروم.

## 1. التحديث إلى Manifest V3
- **الدافع التقني**: تماشياً مع متطلبات متجر كروم (Chrome Web Store) المُلزمة بانتقال جميع الإضافات لـ Manifest V3 لزيادة الخصوصية والأمان واستمرار عمل الإضافة.
- **التطبيق التقني**:
  - تم استبدال صفحات الخلفية (Background pages) التقليدية بـ `Service Workers` في ملف `background.js`.
  - تم احتواء وتشغيل الصوت باستخدام مستند معزول (Offscreen Document) عبر `offscreen.js` و `offscreen.html` لأن تقنية `Service Workers` لا تدعم الـ DOM أو تشغيل ملفات الـ Audio مباشرة.
  - تم طلب الأذونات اللازمة (`storage`, `offscreen`) ضمن `manifest.json`.

## 2. التكامل مع MP3Quran API V3
- **التطبيق**: بدلاً من الاعتماد على ملفات JSON ثابتة تحتاج للتحديث اليدوي (مثل `radios.json` السابقة)، تتصل الإضافة بشكل مباشر وتلقائي بخوادم الموقع الرسمي `mp3quran.net` لجلب أحدث الإذاعات، القراء، الروايات والمصاحف، والسور.
- **لوجستيات الأداء**: تمت برمجة نظام "تخزين محلي مؤقت" (Caching) يحفظ المعلومات ضمن `chrome.storage.local` لتجنب البطء في تحميل الواجهات ولإراحة الخادم من الطلبات المتكررة، مع توفير ميزة جلب احتياطي (Fallback Fetch) إذا فشل الكاش.

## 3. معمارية خالية من المكتبات (Vanilla JS)
- **التطبيق**: تم الاستغناء نهائياً وحذف مكتبة jQuery (`jquery.min.js`)، وإعادة كتابة كافة أسطر الإضافة باستخدام الجافاسكربت النقي (Vanilla JS) الحديث المعتمد على (Async/Await, Promises, Arrow Functions, Fetch API).
- **النتيجة**: حجم أصغر، سرعة استجابة أعلى، تصفح أسرع، وأمان يتوافق مع سياسات الـ CSP الصارمة.

## 4. نظام التشغيل المتقدم للصوتيات
- **المزامنة الحية (State Sync)**: لضمان تزامن واجهة الـ Popup مع حالة الصوت في الـ Offscreen، يتم تبادل الرسائل (Message Passing) وعمل فحص مزدوج (`get_status`) عند كل فتح للنافذة لاستعادة حالة التشغيل أو التوقف دون فقدان آخر عنصر تم تحديده.
- **التشغيل المتتالي (Auto-Next)**: ميزة الانتقال التلقائي للآية/السورة التي تليها عند انتهاء الحالية؛ بُنيت ببرمجة خاصة تتعقب ترتيب قائمة سور القارئ (Surah List) المسترجعة من الـ API وتنتقل للسورة التالية بصمت تام.
- **التكرار (Repeat)**: إعادة السورة الحالية عدداً معيناً من المرات يحددها المستخدم قبل إيقافها أو الانتقال للتالية.
- **تخطي الأخطاء (Auto-Skip)**: برمجة استماع لحدث الأخطاء (`audio.onerror`) في الـ Offscreen Player لتخطي الروابط المعطوبة أو المنقطعة من السيرفر والانتقال للملف التالي فوراً للحفاظ على استمرارية الإذاعة.

## 5. واجهات الإعدادات والمفضلة (Options & Popup)
- **الأداء**: تم استخدام `DocumentFragment` عند بناء المئات من العقد الوهمية (DOM Nodes) المعبرة عن قوائم القراء، مما جعل عملية عرض القوائم الطويلة سلساً وفورياً ودون تجميد للمتصفح (UI Freezing).
- **الفرز الأبجدي والإزالة من التكرار**: معالجة ردود الـ API وفرزها أبجدياً بدقة حسب لغة العرض الحالية، مع برمجة "فلتر" متطور يستبعد تكرار ظهور نفس القارئ بسبب تعدد نسخه على السيرفر الرسمي.
- **قائمة مخصصة (My List)**: حفظ المفضلة باستخدام المعرفات الرقمية المُجردة (IDs) وبنية لامركزية لبناء نظام استدعاء مرن لا يتأثر بتغيير تفضيلات المستخدم اللغوية. تم إصلاح خلل أبعاد الـ Popup الشهير في كروم عن طريق ضبط أطوال CSS قسرية `height: 550px`.

## 6. تعدد اللغات الشامل الديناميكي (Dynamic i18n)
- **التطبيق**: الإضافة بُنيت لدعم أي عدد من اللغات مستقبلاً بسهولة تامة (حالياً: عربي، إنجليزي). ويتم جلب حزم الكلمات (Dictionaries) وقوائم الـ API المطابقة للغة المحددة فوراً من الـ Storage من خلال مفاتيح متغيرة (Dynamic Keys) مثل `api_ar_radios` أو `api_en_reciters`، مع تغيير منطق الواجهة واتجاه العرض (RTL/LTR) جذرياً بدون إعادة تحميل قسري للنافذة الرئيسية وإنما بـ Reload موضعي.

## 7. التصميم العصري (Islamic Premium Theme)
- خطوط Google Fonts الاحترافية (Amiri & Tajawal) أصلية تم استيرادها لتقديم نصوص قرآنية أصيلة وقوائم عصرية مقروءة، وتم تبييض تصريحات الأمان (CSP) داخل الـ `manifest.json`.
- نظام الألوان الإسلامي الزمردي الراقي (`#047857`) مع لمسات ذهبية غنية (`#d97706`).
- دعم أوتوماتيكي وممتاز للوضع الليلي المفضل لدى الأنظمة الحديثة بلون كحلي عميق (Dark Navy `slate-900`) بدلاً من الأسود الميت، لمزج الألوان بجمالية وتقديم تجربة استخدام فخمة ومريحة للعيون.
