## نظرة عامة

**Quran Radio v2.0** هو إعادة بناء شاملة من الصفر لإضافة كروم القديمة. تم الانتقال من معمارية Manifest V2 القديمة القائمة على jQuery وملفات JSON ثابتة، إلى معمارية حديثة متكاملة تعتمد على:

- **Manifest V3** (المعيار الإلزامي الجديد من جوجل)
- **React 19 + Vite** كإطار عمل للواجهات
- **Zustand** لإدارة الحالة العامة
- **MP3Quran API V3** للحصول على البيانات الحية
- **Tailwind CSS** للتنسيق

## الميزات الكاملة

### 🎙 تشغيل الإذاعات الحية (Live Radio)
- بث مباشر لعشرات الإذاعات القرآنية من `mp3quran.net`
- تحديث تلقائي للقائمة عبر API دون أي تعديل يدوي

### 📖 تشغيل المصاحف (Surah Player)
- تشغيل السور القرآنية بأصوات القراء المختلفين
- دعم آلاف القراء والروايات من قاعدة البيانات الرسمية
- اختيار السورة المراد البدء منها بشكل يدوي
- تخزين موضع التشغيل تلقائياً للاستئناف لاحقاً

### ⏭ التشغيل التلقائي المتتابع (Auto-Next)
- انتقال تلقائي للسورة التالية فور انتهاء الحالية
- يتبع ترتيب قائمة السور المتاحة لكل مصحف على الخادم الرسمي
- يعود للبداية تلقائياً بعد آخر سورة (Loop)
- قابل للتفعيل/الإيقاف من واجهة الـ Popup

### 🔁 التكرار (Repeat)
- تكرار السورة الحالية عدداً محدداً من المرات
- حقل عددي للتحكم في عدد مرات التكرار مباشرة من الـ Popup
- يعمل بالتوازي مع منطق Auto-Next: التكرار أولاً ثم الانتقال

### ⚡ استئناف التشغيل (Resume Playback)
- يُحفظ موضع التشغيل الحالي (currentTime) كل 5 ثوانٍ
- عند إعادة اختيار نفس المصحف، يستأنف من حيث توقف

### 📋 القائمة المفضلة الشخصية (My List)
- إضافة أي إذاعة أو قارئ إلى قائمة المفضلة الشخصية
- القائمة تظهر مباشرة في الـ Popup لوصول سريع
- تخزين المفضلات باستخدام المعرفات الرقمية (IDs) فقط، لتجنب مشاكل تغيير اللغة

### 🌐 دعم متعدد اللغات (i18n)
- **6 لغات مدعومة**: العربية، الإنجليزية، الفرنسية، التركية، الفارسية، الإسبانية
- تغيير اللغة فوري من الـ Popup أو صفحة الإعدادات
- تغيير اتجاه الواجهة تلقائياً (RTL/LTR) بدون إعادة تحميل
- جلب أسماء القراء والسور باللغة المختارة من الـ API تلقائياً

### 🎨 الوضع الليلي والنهاري (Dark/Light Theme)
- تحويل كامل للواجهة بين الوضعين من صفحة الإعدادات
- يُحفظ التفضيل ويُطبَّق على كل من الـ Popup والـ Options

### 🔔 إشعارات التشغيل (Toast Notifications)
- إشعار مرئي عند إعادة المحاولة بعد فشل تحميل الصوت (مع العداد)
- إشعار خطأ نهائي بعد 5 محاولات فاشلة متتالية

### 🔍 البحث في القوائم
- حقل بحث فوري لتصفية القراء والإذاعات في صفحة الإعدادات

## التصميم والواجهات

### الـ Popup (نافذة الإضافة الرئيسية)
- **الأبعاد**: `360 × 600px` (ثابتة بـ CSS لتجنب خلل كروم بالأبعاد الديناميكية)
- **الهيدر**: تدرج لوني إسلامي من `primaryDark` إلى `primary` مع حافة ذهبية
- **عنوان الهيدر**: اسم المحطة/السورة الحالية بخط Amiri الرشيق
- **ترا عنوان**: اسم القارئ والمصحف بحجم صغير
- **زر التشغيل**: دائري بظل ذهبي مزدوج مع تأثير hover وactive سلس
- **أدوات السورة**: لوحة شفافة تظهر فقط للقراء (تعمى وتُعطَّل للإذاعات)
- **قائمة المفضلة**: مع أيقونة 📻 للإذاعات و 📖 للقراء، والعنصر المحدد يتميز بلون ذهبي
- **الفوتر**: يحتوي زر الإعدادات + منتقي اللغة

### صفحة الإعدادات (Options)
- تصميم ثلاثي الأعمدة: **الإذاعات | القراء | قائمتي**
- ارتفاع ثابت للأعمدة `70vh` مع تمرير داخلي
- شريط بحث في كل عمود
- عداد عدد العناصر في "قائمتي"
- تغيير اللغة والثيم من الهيدر مباشرة

### لوحة الألوان
```
primary:      #047857  (أخضر إسلامي)
primaryDark:  #065f46  (أخضر داكن)
primaryLight: #10b981  (أخضر فاتح)
gold:         #d97706  (ذهبي دافئ)
bgDark:       #0f172a  (كحلي عميق)
bgLight:      #f8fafc  (أبيض رمادي)
cardDark:     #1e293b  (رمادي مزرق)
```

### الخطوط
- **Amiri** — للعناوين والنصوص القرآنية (serif كلاسيكي)
- **Tajawal** — للنصوص العربية الحديثة في القوائم

## المعمارية التقنية

### Service Worker (`background.js`)
مركز التنسيق الرئيسي للإضافة. يستقبل الرسائل من الـ Popup والـ Offscreen ويتخذ القرارات:

| الرسالة | الوظيفة |
|---|---|
| `play_radio` | تشغيل رابط إذاعة حي |
| `play_surah` | تشغيل سورة بمعلوماتها وقائمة سورها |
| `stop` | إيقاف التشغيل وتحديث الـ Badge |
| `sync_time` | تحديث موضع التشغيل في Storage |
| `status` | استعلام عن حالة التشغيل الفعلية |
| `track_ended` | منطق التكرار أو Auto-Next عند انتهاء الصوت |

**الـ Badge**: نقطة صغيرة خضراء مع رمز ▶ عند التشغيل، وتختفي عند الإيقاف.

### Offscreen Document (`offscreen.js`)
المشغّل الصوتي الحقيقي. يعيش خارج الـ Service Worker لأن الأخير لا يدعم DOM/Audio:

- **إنشاء عنصر Audio** ديناميكياً وتنظيف الأحداث بالكامل عند كل تشغيل جديد
- **معالجة الأخطاء**: 5 محاولات إعادة اتصال قبل الاستسلام (errorCode 4)
- **Stalled Detection**: إذا توقف الصوت 10 ثوانٍ، يحاول إعادة الاتصال تلقائياً
- **إيقاف نظيف للراديو**: تدمير عنصر الـ Audio كلياً عند الإيقاف لمنع الـ Buffering الخلفي
- **Sync دوري**: يرسل `currentTime` كل 5 ثوانٍ للـ background لحفظه

### State Management (`playerStore.js`)
Zustand Store يجمع كل الحالة:
- `playbackState` — حالة التشغيل الحالية (كاملة من Storage)
- `userList` — القائمة المفضلة الشخصية
- `userLang` — اللغة الحالية
- `userTheme` — الثيم الحالي

يستمع لتغييرات `chrome.storage.onChanged` ليتزامن فورياً مع أي تغيير يحدث في الـ background.

### نظام الكاش (Caching)
كل بيانات API تُخزَّن في `chrome.storage.local` بمفاتيح ذكية:
```
api_ar_radios    → قائمة الإذاعات بالعربية
api_en_reciters  → قائمة القراء بالإنجليزية
api_ar_suwar     → أسماء السور بالعربية
...إلخ
```
عند تغيير اللغة، تُجلب البيانات الجديدة وتُخزَّن بالمفتاح الخاص بها.

### Vite Build (متعدد المداخل)
```js
// vite.config.js
input: {
  popup: 'popup.html',
  options: 'options.html',
  background: 'src/background/background.js',
  offscreen: 'src/offscreen/offscreen.js'
}
```

## نظام التدويل i18n

### ملفات الترجمة
تقع في `public/_locales/<lang>/messages.json` بصيغة Chrome i18n القياسية:
```json
{
  "extName": { "message": "إذاعة القرآن الكريم" },
  "surahLabel": { "message": "سورة" },
  "playbackRetry": { "message": "إعادة الاتصال... المحاولة $1" }
}
```

### اللغات المدعومة
| الكود | اللغة | الاتجاه |
|---|---|---|
| `ar` | العربية | RTL |
| `en` | English | LTR |
| `fr` | Français | LTR |
| `tr` | Türkçe | LTR |
| `fa` | فارسی | RTL |
| `es` | Español | LTR |

### آلية العمل
1. **`useI18n(lang)`** — Hook مخصص يجلب الترجمات من ملف JSON المقابل
2. **الـ Background**: يستخدم `getTranslation()` المستقلة لقراءة الترجمات (لا تعتمد على React)
3. **API**: تُضاف لاحقة `?language=<lang>` لكل طلب (مع تحويل `en` → `eng`)
4. **RTL/LTR**: يُطبَّق على `document.documentElement.dir` فوراً عند تغيير اللغة

## الأخطاء التي تم إصلاحها

### 🐛 خلل أبعاد الـ Popup في كروم
**المشكلة**: كروم يُغيّر أبعاد الـ Popup بشكل عشوائي بسبب المحتوى الديناميكي.  
**الحل**: تحديد `width: 360px; height: 600px` ثابتة على العنصر الجذر بـ CSS.

### 🐛 خلل Auto-Next مع متغير غير معرّف
**المشكلة**: `ReferenceError` في الـ background عند محاولة استخدام `surahLabel` قبل تحميله من الترجمة.  
**الحل**: نقل استدعاء `getTranslation()` قبل أي استخدام للمتغير وانتظار الـ Promise بـ `await`.

### 🐛 تكرار القراء في قائمة الـ Options
**المشكلة**: API يُعيد نفس القارئ عدة مرات (مرة لكل رواية).  
**الحل**: استخدام `Map` مع معرف القارئ (`id`) لإزالة التكرار قبل العرض.

### 🐛 عدم تزامن اللغة بين الـ Popup وصفحة الإعدادات
**المشكلة**: تغيير اللغة في أحد النوافذ لا يُحدّث الأخرى.  
**الحل**: الاستماع لـ `chrome.storage.onChanged` في الـ Store وتحديث `userLang` تلقائياً.

### 🐛 استمرار الـ Buffering بعد إيقاف الراديو
**المشكلة**: بعد الضغط على إيقاف، يستمر الراديو في تحميل البيانات في الخلفية.  
**الحل**: عند إيقاف البث الحي (`duration === Infinity`)، يُزال عنصر الـ Audio من الـ DOM كلياً وتُصفَّر مصادره.

### 🐛 فشل الـ Seek عند استئناف التشغيل
**المشكلة**: محاولة تحديد `currentTime` قبل أن يكون الصوت جاهزاً يؤدي لتجاهل الموضع.  
**الحل**: استخدام listener على حدث `timeupdate` للانتظار حتى يبدأ الـ Audio engine فعلياً، ثم تحديد الإحداثية.

### 🐛 أسماء القراء لا تتناسب مع اللغة في الـ Popup
**المشكلة**: القائمة المفضلة تُحفظ الأسماء بلغة التسجيل فقط.  
**الحل**: بناء نظام "Hydration" يُعيد جلب الأسماء المترجمة من الكاش/API عند كل تغيير للغة.

### 🐛 خطأ اتصال عند انتهاء بث الراديو بسبب انقطاع الشبكة
**المشكلة**: `audio.onerror` لا يُميّز بين أنواع الأخطاء، فيتم معالجة قطع المستخدم للبث كخطأ حقيقي.  
**الحل**: فحص `newAudio.error.code === 1` (user abort) وتجاهله قبل أي معالجة.

## بناء المشروع وتشغيله

### متطلبات التشغيل
- Node.js >= 18
- npm >= 9

### تثبيت التبعيات
```bash
npm install
```

### وضع التطوير
```bash
npm run dev
```
> ملاحظة: وضع التطوير لا يولّد ملفات الإضافة المباشرة. يُستخدم للتحقق من الكود فقط.

### بناء الإضافة للنشر
```bash
npm run build
```
الملفات الجاهزة ستكون في مجلد `dist/`.

### تحميل الإضافة على كروم (Developer Mode)
1. افتح `chrome://extensions`
2. فعّل **Developer Mode** من الأعلى
3. اضغط **"Load unpacked"**
4. اختر مجلد `dist/`
